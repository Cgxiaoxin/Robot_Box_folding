# 轨迹连续插值“防再撞最小修”记录（2026-04-17）

## 背景

在完成“轨迹连续插值 + `duration/hold` 语义改造”后，上机运行出现两次碰撞；同时在按下急停后再次使能，机械臂出现“急速冲撞”的现象。

本次记录用于简要说明：问题链路、最小化防护修复点、以及本地无硬件回归验证方式。

## 风险链路总结（定位结论）

1. **播放前未对齐轨迹首点**
   - 播放直接进入 `point[i] -> point[i+1]` 段插值，如果实际姿态与轨迹首点不一致，会发生“从当前位置抄近路并轨”，路径不可控，容易撞障碍物。

2. **急停后重新使能可能继续追旧目标**
   - 急停主要停了软件播放线程，但底层/SDK 侧可能仍保留最后一次/最近几次 `move_j(blocking=False)` 的目标。
   - 重新使能仅 `enable()`，没有显式用“当前姿态保持目标”覆盖旧目标，因此出现上电后快速追目标的风险。

3. **轨迹播放路径绕过软限位校验**
   - 手动 `set_position` 有软限位保护，但轨迹播放的 `move_j` 下发路径原本没有限位检查，脏数据会直接被下发。

4. **stream 录制 `duration` 归属错位一拍**
   - stream 模式下采样点的 `duration` 实际应代表“上一段耗时”，但播放器消费的是“当前点到下一点的段耗时”，会导致首段时长异常（被钳到最小段时长，动作很猛）及整体节奏漂移。

## 最小修复内容（防再撞）

### 1) 播放前校验并对齐首点

文件：`robot_arm_web/backend/group_controller.py`

- 新增首点偏差阈值参数：
  - `TRAJECTORY_START_ALIGN_TOLERANCE_RAD`：偏差超过该值则需要低速阻塞对齐
  - `TRAJECTORY_START_MAX_DEVIATION_RAD`：偏差超过该值直接拒绝播放
- 在 `play_trajectory()` 启动线程前执行 `_prepare_playback_start(points)`：
  - 预扫描所有点：检查各臂目标是否落在软限位内（发现越界立即拒播）
  - 计算首点与当前姿态最大关节偏差：
    - `> MAX_DEVIATION`：拒绝播放并报错
    - `> ALIGN_TOLERANCE`：切换 `very_slow` 速度，阻塞对齐首点，之后恢复原速度

配置：`robot_arm_web/backend/config.py`

- 新增：
  - `TRAJECTORY_START_ALIGN_TOLERANCE_RAD`（默认 `0.02`）
  - `TRAJECTORY_START_MAX_DEVIATION_RAD`（默认 `0.12`）

### 2) 急停后重新使能先“保持当前姿态”

文件：`robot_arm_web/backend/group_controller.py`

- `emergency_stop()` 后对 `ArmRuntime` 标记 `recovery_hold_required=True`
- `init_arm()` 使能时若检测到 `recovery_hold_required`：
  - 读取当前角度
  - 以 `very_slow` 速度阻塞下发当前角度作为保持目标（覆盖可能残留的旧目标）
  - 再恢复原速度参数，并清除标记

### 3) 轨迹插值下发增加软限位拦截

文件：`robot_arm_web/backend/group_controller.py`

- 在 `_send_arm_targets()` 中新增整组软限位检查：
  - 一旦发现任意关节越界：立即 `stop_playback`（通过 `_stop_playback` event）并报错
  - 通过检查才执行 `move_j(blocking=False)`

### 4) stream 录制 `duration` 归位到“前一段”

文件：`robot_arm_web/backend/drag_teach_service.py`

- stream 模式下新增点时：
  - 当前点 `duration` 保持 `0.0`
  - 将“本次采样间隔”写回上一点的 `duration`（并同步 `delay` 兼容字段）

### 5) 轨迹引擎兼容与时长计算修正

文件：`robot_arm_web/backend/trajectory_engine.py`

- `TrajectoryPoint` 增加 `delay` 兼容入参（`InitVar`），仍可用旧字段构造
- `calculate_duration()` 不再把最后一个点的 `duration` 计入运动时长（播放器末点不执行段运动，只执行可选 `hold`）

- `robot_arm_web/tests/test_group_controller_logic.py`
  - 首点偏差过大拒播
  - 首点偏差在允许范围内可低速对齐
  - 急停后重新使能触发“保持当前姿态”覆盖逻辑
- `robot_arm_web/tests/test_drag_teach_service.py`
  - stream 模式 `duration` 写回上一点
- `robot_arm_web/tests/test_trajectory_engine.py`
  - `delay` 兼容构造与加载
  - 时长计算与新语义一致

运行命令：

```bash
./robot_arm_web/tests/run_tests.sh
```

## 上机建议（安全操作）

- 首次验证请在空载、远离障碍物、`very_slow` 速度下进行。
- 验证两条关键安全性：
  1. 当前姿态明显偏离轨迹首点时，是否会被正确拒播并报错；
  2. 急停后重新使能，是否只原地保持，不再追旧轨迹目标。
